## Rutine for kvalitetskontroll
<span class='underline'>Før hver merge til master, skal følgende punkter gjennomføres: </span> 

1. Følge KVIB sine konvensjoner for CSS-klassenavn. Se [Om KVIB -> Konvensjonerfor CSS-klassenavn](./infoKvib.mdx) for beskrivelse.


2. Teste komponenter med skjermleser:
    - NVDA


3. Teste komponenter med browserverktøy:
    - Wave for UU-testing
    - Lighthouse for beste praksis
    - Axe


4. Teste i ulike browsere:
    - Chrome / chromium
    - Firefox
    - Safari


5. Teste med 400% zoom på desktop, samt teste på små skjermer.


6. QA på all ny kode.


7. Oppdater change-log (Keep a Changelog )


8. Følge regler for dokumentasjonssiden:
    - Struktur
    - Obligatorisk innhold
